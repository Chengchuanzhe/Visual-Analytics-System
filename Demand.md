# 实验 2：全球海洋-大气耦合时空可视分析系统

## 项目概览

本项目为可视分析课程实验 2，在实验 1 基础绘图与交互能力之上，聚焦**高级图表设计开发 + 视觉编码设计优化 + 选题场景适配 + 系统前端界面原型搭建**。

- **选题**：全球海洋-大气耦合时空可视分析系统
- **周期**：第 4–10 周（共 7 周）
- **团队**：4 人小组
- **技术栈**：D3.js（图表核心）、前端静态原型（HTML/CSS/JS）

---

## 实验目标

1. **图表开发**：掌握饼图、散点图等基础图表，以及 ≥1 类适配选题的高级图表（桑基图 / 弦图），理解时空数据适配场景
2. **数据统筹**：数据维度 ≥14 维、规模 ≥3000 条、跨 3 年以上、覆盖 ≥6 个地域空间
3. **视觉编码优化**：色盲友好配色、对比度优化、格式塔原则，制定团队视觉规范
4. **选题场景适配**：新颖图表需求分析与原型设计，1 名成员主责代码实现
5. **前端原型搭建**：多图表组合逻辑，≥4 类图表组合，静态代码搭建含后续交互接口
6. **工程化思维**：代码复用、注释规范、可维护性与可拓展性

## 周次进度安排（第 4–10 周）

### 第 4 周：分析任务抽象 + 数据统筹 + 基础图表开发

**实验内容**

- **分析任务抽象**（全员）：基于海洋-大气耦合主题，小组讨论明确 3–5 个核心时空分析任务（如"污染源占比分析""多维度指标关联分析""时空趋势分析"），形成任务清单
- **数据统筹**（主责 A + 全员）：
  - 主责 A：确定数据维度（≥12 维）、数据来源、缺失值/异常值处理策略，制定数据采集计划
  - 全员：采集跨 3 年以上、6 个及以上地域的时空数据，完成数据清洗（去重、补全、格式统一）
  - 主责 A：整合数据，转换为 JSON 格式，生成"实验 2 选题扩展数据集"
- **基础图表开发**（主责 A/B + 全员）：
  - 主责 A：独立开发饼图（含图例、数据标签、基础悬浮交互）
  - 主责 B：独立开发散点图（含坐标轴映射、颜色编码、基础悬浮交互）
  - 全员：互相审核图表功能完整性，测试数据映射准确性

**必提交成果**
1. 小组共同：选题分析任务清单、实验 2 选题扩展数据集（JSON 格式）
2. 主责 A：数据抽象说明书（含 ≥12 维维度、数据来源、处理策略，标注 ≥3000 条数据量）、饼图截图（含悬浮交互）
3. 主责 B：散点图截图（含悬浮交互）
4. 全员：数据处理过程截图（每人 1 张，标注参与环节）、图表审核意见文档

  - 主责 C：汇总所有图表的优化结果，制定图表基础视觉规范（字体、字号、间距基础要求）

**必提交成果**

## 视觉规范要求

- **色盲友好配色**：避免红-绿、蓝-紫冲突，推荐 ColorBrewer 或 Okabe-Ito 色板
- **对比度**：文字与背景对比度 ≥4.5:1（WCAG AA 标准）
- **格式塔原则**：接近性、相似性、连续性，提升大数据量下信息辨识度
- **全系统一致性**：统一颜色表（含 RGB/十六进制值）、字体、字号、间距、边框、图例、轴标签样式
- **验证工具**：Color Safe（色盲模拟）、colorBrewer（颜色表制定）

## 重点与难点提示

**重点**
- 多类图表核心逻辑：基础图表数据映射规则、高级图表（桑基图/弦图）的流/关联表达逻辑，所有成员需掌握核心实现
- 选题数据统筹：所有成员参与跨 3 年以上、6 个地域空间的时空数据采集、清洗与规范，确保数据维度 ≥14 维、规模 ≥3000 条
- 视觉编码优化：小组协同制定视觉规范，所有成员参与图表颜色/布局优化，实现色盲友好配色与全系统视觉一致性
- 选题适配与新颖图表：小组共同分析场景痛点，设计新颖图表原型，1 名成员主责代码实现，确保创新与可行性平衡
- 多图表组合与原型搭建：实现 ≥4 类图表的组合整合与前端静态原型搭建，预留后续交互接口

**难点**
- 复杂图表数据适配：不同图表数据格式差异大，需协同完成选题时空数据的格式转换，确保数据映射准确
