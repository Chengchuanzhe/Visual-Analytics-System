1.3 审计发现

发现 1（严重）：成员 A 的饼图使用了自定义颜色值 (#0077BB, #33BBEE, #009988, #EE7733, #CC3311, #EE3377)， 而非团队统一的 Okabe-Ito 色板。其中 #EE3377（赤道区域·粉红） 与 #CC3311（南极·深红）在红绿色盲模拟下几乎不可区分。

发现 2（中等）：Okabe-Ito 黄色 #F0E442（印度洋） 在白底上的对比度仅 1.4:1，低于 WCAG AA 标准的 4.5:1。 建议：在图例/标签中为黄色元素添加 1px 深色描边，或改用 #CCBB00 作为印度洋的替代色（保持暖色调但提高对比度）。

发现 3（轻微）：天蓝 #56B4E9（北极）与 蓝 #0072B2（太平洋）在蓝色弱模拟下虽有差异但不够显著， 建议增大色相差距或添加不同线型/填充图案作为双重编码。

2. 色盲友好配色方案（3 套备选）

方案 A：Okabe-Ito 标准版（当前采用）

优点：学术界公认、8色完备、大部分色盲类型可区分。
缺点：黄色对比度低、蓝/天蓝在蓝色弱下接近。

方案 B：ColorBrewer Dark2（备选）

优点：ColorBrewer 官方色盲验证、低饱和度更耐看。
缺点：仅 8 色（够用但无扩展空间）、部分颜色语义感弱。

方案 C：Okabe-Ito 增强版（推荐替代）

优点：保留 Okabe-Ito 结构，替换黄色为琥珀色 #D4A017（对比度 3.2:1）， 区分太平洋蓝 #005A8B 与北极天蓝 #4DB8E8。
缺点：丢失原版 Okabe-Ito 的"橙色-黄色"逻辑连续性，但对色盲更友好。

3. 最终选定方案与整改清单

最终决定：采用方案 A（Okabe-Ito 标准版）作为主色板。

理由：① 已有 5/6 图表使用 Okabe-Ito；② 学术界认可度高； ③ 黄色对比度问题通过在黄色元素上加 1px #999 描边解决，无需换色。
