1. 交互接口说明

1.1 原型架构

架构模式：导航栏 + 侧边筛选区 + 图表网格 + 底部状态栏。 图表通过 iframe 嵌入独立 HTML 文件（保持各图表独立运行能力）。 筛选区通过 postMessage 向所有图表广播筛选事件。

1.2 Dashboard → Chart 消息协议

// 从原型向 iframe 中的图表发送筛选消息
{
  "event":     "filter:change",   // 事件类型
  "type":      "region",            // 筛选维度: region | date | dimension
  "value":     "太平洋",             // 筛选值
  "hidden":    false,                // 是否隐藏
  "timestamp": 1714400000000        // 事件时间戳
}

1.3 Chart → Dashboard 消息协议（预留）

// 图表向原型发送就绪/选中/刷选事件
{
  "event":  "chart:ready",         // 图表渲染完成
  "chart":  "parallel-coords",   // 图表标识
  "records": 720                  // 渲染记录数
}

{
  "event":  "chart:brush",         // 图表内刷选事件
  "chart":  "parallel-coords",
  "dimension": "sst",
  "range":  [15, 28]
}

1.4 Dashboard 全局 API

POST Dashboard.filterByRegion(region, hidden) 筛选指定海域，向所有图表广播
POST Dashboard.resetAllFilters() 重置所有筛选条件
POST Dashboard.switchView(view) 切换视图: "overview" | "analysis" | "flow" | "trend"
GET Dashboard.getChartPaths() 获取所有图表 HTML 的路径映射

1.5 各图表监听示例（供图表开发者参考）

// 在任意图表 HTML 的 JS 中添加以下代码即可响应筛选：
window.addEventListener("message", (event) => {
  if (event.data.event === "filter:change") {
    const { type, value, hidden } = event.data;
    if (type === "region") {
      if (hidden) {
        hiddenRegions.add(value);
      } else {
        hiddenRegions.delete(value);
      }
      render();  // 重新渲染图表
    }
  }
});

// 图表就绪后通知 Dashboard
window.parent.postMessage({
  event: "chart:ready",
  chart: "pie",
  records: 3600
}, "*");
