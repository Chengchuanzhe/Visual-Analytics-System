#### 2. 高级图表开发 - 桑基图

**需求分析**
- 展示对象：海表温度 → 海域流向 + 盐度变化
- 数据结构：节点(海表温度分档 × 海域) + 边(流向关系)
- 交互：悬浮高亮流向、点击筛选

**数据转换流程**

Step 1: 温度分档
```javascript
// 将连续温度分为5个档次
const temperatureBins = [
  {min: -2, max: 5, label: '极冷'},
  {min: 5, max: 12, label: '冷'},
  {min: 12, max: 20, label: '温'},
  {min: 20, max: 26, label: '温暖'},
  {min: 26, max: 32, label: '热'}
];

const getBinLabel = (temp) => {
  return temperatureBins.find(b => temp >= b.min && temp < b.max).label;
};
```

Step 2: 节点生成
```javascript
// 节点 = 温度分档 × 海域
const nodes = {};
data.forEach(d => {
  const key = `${getBinLabel(d.sst)}-${d.region}`;
  if (!nodes[key]) {
    nodes[key] = {
      id: key,
      name: `${getBinLabel(d.sst)}(${d.region})`,
      value: 0
    };
  }
  nodes[key].value++;
});
```

Step 3: 边生成(流向关系)
```javascript
// 按时间序列相邻记录生成边
const edges = [];
const sortedData = data.sort((a, b) => new Date(a.time) - new Date(b.time));

for (let i = 0; i < sortedData.length - 1; i++) {
  const source = `${getBinLabel(sortedData[i].sst)}-${sortedData[i].region}`;
  const target = `${getBinLabel(sortedData[i+1].sst)}-${sortedData[i+1].region}`;
  
  const edgeKey = `${source}->${target}`;
  if (!edges[edgeKey]) {
    edges[edgeKey] = {source, target, value: 0};
  }
  edges[edgeKey].value++;
}
```

**节点定位算法**

使用改进的重力模型：
- 节点高度 = 源点所有流向的总值
- 垂直位置 = 根据流向值平衡分布
- 水平位置 = 固定分层布局

// 计算节点纵坐标
const nodeHeight = {};
nodes.forEach(node => {
  nodeHeight[node.id] = outgoingEdges[node.id]

// 分层布局

### 关键决策记录

**决策1: 雷达图维度选择**
- 背景: 需要展示多维特征
- 方案: 选择6个关键维度(温度、盐度、风、压力、降水、湿度)
- 理由: 覆盖海洋-大气耦合的核心特性

**决策2: 桑基图数据分档**
- 背景: 流向可视化需要离散化处理
- 方案: 温度分5档(极冷/冷/温/温暖/热)
- 理由: 易于理解，避免节点过多

**决策3: 流向追踪方法**
- 背景: 需要表示时间序列中的温度变化
- 方案: 按时间排序后相邻记录间连接
- 理由: 真实反映实际的热量流向过程

### 关键经验

1. **数据聚合很关键**: 3600条原始数据直接渲染会很慢，通过合理的数据聚合可以显著提升性能
2. **编码方案要清晰**: 每个视觉通道的编码含义必须明确，避免认知冲突
3. **性能测试要提早**: 大数据量下的性能问题需要提前发现并优化

  const yScale = d3.scaleLinear()
    .domain(d3.extent(data, d => d[yField]))
    .range([height - margin.bottom, margin.top]);
  
  return {xScale, yScale};
}
