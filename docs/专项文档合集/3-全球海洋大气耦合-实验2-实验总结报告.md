## 1. 项目概述

### 1.1 背景与意义

全球气候变化是当今世界面临的重要科学问题之一。海洋与大气的相互作用对全球气候具有深远的影响。本项目通过交互式数据可视化手段，揭示全球不同海域的海洋-大气耦合关系及其时空演变规律。

**关键科学问题**:
- 全球不同海域的海洋-大气特征如何?
- 这些特征在时间上如何变化?
- 不同海域之间存在什么关联?
- 海洋温度、大气压等参数如何相互影响?

**可视化目标**:
- 支持多维数据的高效可视化
- 揭示数据中的隐含模式
- 提供交互式数据探索能力
- 辅助科学决策

### 1.2 项目范围

**数据范围**:
- 时间跨度: 3年 (2020-2022年)
- 地理范围: 全球6个主要海域/地区
- 数据粒度: 月度数据
- 数据规模: 3600条数据记录

**系统功能**:
- 5+种可视化图表类型
- 4个独立的数据分析模块
- 响应式交互界面
- 无缝的多图表集成

**成果形式**:
- 交互式数据可视化系统
- 数据分析报告
- 技术文档与代码

## 3. 可视化设计

### 3.1 可视化图表概览

系统包含5个核心可视化图表:

#### 3.1.1 饼图(圆形百分比图)

**作用**: 展示各海域在总数据中的比例分布

```
       太平洋
        /    \
       /      \ 
   大西洋 -- 印度洋
       \      /
        \    /
       北极-南极
```

**编码方式**:
- 角度 → 海域占比
- 颜色 → 海域类别 (Viridis色板)
- 大小 → 占比大小

**交互功能**:
- 悬浮显示数值
- 点击突出显示
- 分层动画

**数据映射**:
```
数据 → 统计 → 求和 → 归一化 → 角度转换 → 弧形
```

---

#### 3.1.2 气泡图(四维散点图)

**作用**: 展示4个维度的关联关系(通常是温度、压力、风速、湿度)

**编码方式**:
- X轴 → 海表温度
- Y轴 → 气压
- 气泡大小 → 风速
- 气泡颜色 → 海域
- 时间 → 动画序列

**交互功能**:
- 拖拽探索
- 时间滑块控制
- 维度切换

**优势**: 
- 在二维平面上展示4维数据
- 支持多参数关系分析

---

#### 3.1.3 散点图(基础两维关系)

**作用**: 展示海表温度与大气压的关系

**编码方式**:
- X轴 → 海表温度 (℃)
- Y轴 → 大气压 (hPa)
- 点颜色 → 海域
- 点大小 → 时间序列

**交互功能**:
- 拖拽缩放
- 点击查询详细信息
- 过滤显示

**数据分布特征**:
- 相关系数: 0.68 (中等正相关)
- 分布形态: 聚类明显

---

#### 3.1.4 雷达图(多维指标)

**作用**: 同时展示6个维度的多个指标(温度、压力、湿度、风速、盐度、pH值)

**编码方式**:
- 轴线 → 各维度指标
- 轴线长度 → 标准化指标值
- 颜色 → 海域类别
- 透明度 → 时间变化

**优势**:
- 一次展示6个维度
- 易于比较多个对象
- 视觉冲击强

**使用场景**:
- 区域特征对比
- 时间序列变化
- 异常值检测

---

#### 3.1.5 桑基图(流向关系)

**作用**: 展示海域间的能量/物质流动关系

**编码方式**:
- 节点 → 海域 (左侧源+右侧目标)
- 连接线宽度 → 流量大小
- 颜色 → 源海域
- 透明度 → 流动强度

**流向说明**:
```
暖流 (源) ──→ 冷流 (目标)
         ──→ 中温流
         ──→ 季风流
```

**交互功能**:
- 悬浮高亮流向
- 点击显示详情
- 动画演示流动

---

#### 3.1.6 平行坐标图(新颖图表)

**作用**: 展示多达8-10个维度的高维数据

**编码方式**:
- Y轴线 → 各维度
- 折线 → 单条数据记录
- 颜色 → 海域类别
- 线条宽度 → 权重/重要度

**特点**:
- 支持高维数据展示 (8+维)
- 支持维度重排序
- 支持范围筛选
- 异常值易检测

**优势**:
- 最多维数据展示(相比其他图表)
- 支持动态交互
- 揭示数据模式

### 4.3 核心代码示例

#### D3.js 饼图实现

```javascript
// 数据准备
const data = [
  {region: "太平洋", value: 1200},
  {region: "大西洋", value: 800},
  {region: "印度洋", value: 600}
];

// 颜色映射
const colorScale = d3.scaleOrdinal()
  .domain(data.map(d => d.region))
  .range(["#0173B2", "#DE8F05", "#029E73"]);

// 饼图生成器
const pie = d3.pie()
  .value(d => d.value);

const arc = d3.arc()
  .innerRadius(0)
  .outerRadius(200);

// SVG容器
const svg = d3.select("#chart")
  .append("svg")
  .attr("width", 500)
  .attr("height", 500)
  .append("g")
  .attr("transform", "translate(250, 250)");

// 绘制饼图
svg.selectAll("path")
  .data(pie(data))
  .enter()
  .append("path")
  .attr("d", arc)
  .attr("fill", d => colorScale(d.data.region))
  .on("mouseover", function() {
    d3.select(this).transition().attr("opacity", 0.8);
  })
  .on("mouseout", function() {
    d3.select(this).transition().attr("opacity", 1);
  });
```

### 5.4 数据分析成果

**关键发现**:

1. **地域特征分化明显**
   - 太平洋: 温度范围18-28℃, 压力1010-1015hPa
   - 北极: 温度范围-5-5℃, 压力980-1020hPa
   - 赤道: 温度范围25-32℃, 压力1008-1012hPa

2. **时间序列变化规律**
   - 夏季(6-8月): 温度最高, 风速最小
   - 冬季(12-2月): 温度最低, 风速最大
   - 春秋(3-5, 9-11月): 过渡期, 变化剧烈

3. **参数相关性**
   - 温度与盐度: -0.45 (负相关)
   - 温度与溶解氧: 0.62 (正相关)
   - 风速与湿度: 0.38 (弱正相关)

4. **异常值分析**
   - 检测到16条异常值 (占0.4%)
   - 主要分布在北极及南极冬季
   - 原因: 极端天气事件或数据传输错误

## 8. 关键问题与解决方案

### 8.1 技术问题

**问题1: 大数据集渲染性能**
- 现象: 3600条数据在散点图中渲染时帧率下降
- 原因: D3.js处理数据量超过阈值
- 解决方案:
  - 采用数据采样(保留关键数据点)
  - 使用Canvas替代SVG(可选)
  - 实现虚拟滚动
- 效果: FPS从30提升到58-60

**问题2: 颜色区分度不足**
- 现象: 6个海域的颜色相似, 难以区分
- 原因: 初期使用简单色板
- 解决方案:
  - 采用Viridis色板导出方案
  - 进行WCAG对比度验证
  - 色盲模拟测试
- 效果: 所有颜色对都满足AA标准

**问题3: 响应式布局困难**
- 现象: 平板和手机上图表显示不完整
- 原因: 固定宽度CSS设计
- 解决方案:
  - 采用CSS Grid响应式布局
  - 实现多断点设计
  - 使用Media Queries
- 效果: 支持3种主要屏幕尺寸

### 8.2 管理问题

**问题1: 模块间接口不清**
- 现象: 各成员开发的图表难以整合
- 原因: 缺乏统一的接口规范
- 解决方案:
  - 制定ChartInterface规范
  - 建立代码审查机制
  - 使用iframe隔离
- 效果: 集成时间从3天降低到1天

**问题2: 工作进度不同步**
- 现象: 有成员提前完成, 有成员延期
- 原因: 任务粒度不明确
- 解决方案:
  - 细化周任务清单
  - 建立每日站会
  - 进度可视化
- 效果: 最终按时交付

**问题3: 文档编写滞后**
- 现象: 代码完成后文档未及时更新
- 原因: 开发与文档脱离
- 解决方案:
  - 采用同步文档策略

  - 定期文档审查
- 效果: 文档完整性提升到95%+
